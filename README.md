Мобильный одностраничный лендинг для продажи лифтинг-маски для шеи. Гео: Узбекистан. ЦА: женщины 35+.

---

## УТП

- **Скидка 50%** — 149 000 ₽ вместо 299 000 ₽
- **2+1 в подарок** — при покупке двух упаковок третья бесплатно
- **Доставка 1–2 дня** по Узбекистану
- **Оплата при получении** — без предоплаты
- **Обмен/возврат 30 дней**

## GitHub Pages

1. **Структура репозитория** — при пуше в GitHub убедитесь, что папка `img/` с файлами (`hero-product.png`, `before.png`, `after.png`) добавлена в репозиторий:
   ```bash
   git add img/
   git status  # проверьте, что img/*.png в списке
   git commit -m "Add images"
   git push
   ```

2. **Регистр имён** — GitHub (Linux) чувствителен к регистру. Файлы должны называться точно: `hero-product.png`, а не `Hero-Product.PNG`.

3. **Внешние фото** (шаги 1–4) — Pexels/Unsplash иногда блокируют загрузку с других сайтов. Добавлен `referrerpolicy="no-referrer"`. Если не помогают — скачайте фото и положите в `img/step1.jpg` … `img/step4.jpg`, затем замените URL в `index.html`.

4. **Базовый путь** — скрипт в `<head>` автоматически подставляет путь для проектных страниц (например, `username.github.io/neck-mask/`).

## Кастомизация

### Изображения
- `img/hero-product.png` — баннер в hero
- `img/before.png` / `img/after.png` — блок До/После
- Шаги 1–4 — внешние URL (Pexels, Unsplash), можно заменить на локальные

### Форма заказа
В `script.js` замените `alert()` на отправку в CRM/сервер (Bitrix24, amoCRM, SendPulse и т.п.):

```javascript
// Пример: отправка в Telegram
fetch('https://api.telegram.org/bot<TOKEN>/sendMessage', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    chat_id: '<CHAT_ID>',
    text: `Заказ: ${name}, ${phone}`
  })
});
```

### Цены
Измените в `index.html`: `299 000` и `149 000` сум.

### Таймер
В `script.js` — `endTime` задаёт время окончания акции. Сейчас: +10 часов от загрузки.
